Cure v0.17.0 is the release where the type system grows up.

For seven versions we have been building infrastructure: a lexer, a
parser, a bidirectional type checker, refinement types, Z3 integration,
FSMs, a protocol system, an effect system. It worked. It even shipped
real programs. But the gap between Cure as a language and Cure as a
*dependently-typed* language was conspicuous. Every time you wanted to
say "this function's return type is `Vector(T, m + n)`", the compiler
nodded politely and moved on.

In v0.17.0 it stops nodding.

## Sigma types, Pi types, equality types

Three shapes that have been missing arrive together:

- **Sigma types** pair a value with a type that depends on it.
  `Sigma(n: Nat, Vector(T, n))` is "a pair of a natural number and
  a vector of exactly that length."
- **Pi types** are function types whose return depends on the
  arguments. `fn append(xs: Vector(T, m), ys: Vector(T, n)) ->
  Vector(T, m + n)` is now *checked*, not hoped.
- **Equality types** let programs carry proofs that two terms are
  equal. `refl(x) : Eq(T, x, x)` is the single constructor;
  `rewrite eq in expr` is the single eliminator. Both vanish at
  runtime.

The glue is a small terminating normaliser, `Cure.Types.Reduce`,
that folds closed type-level arithmetic, booleans, and pair projection.
`Vector(T, 2 + 3)` is the same type as `Vector(T, 5)` -- no SMT call
required.

## Implicit arguments and the unification trace

Implicit arguments land with `{T}` braces:

```cure
fn id({T}, x: T) -> T = x
```

At each call site, first-order unification with an occurs check solves
the implicit from the explicit arguments. When it fails, Cure emits a
`:unification_trace` pipeline event the LSP renders in hover and the
CLI prints in errors. You can see *where* the constraint died.

## Hole-driven development

The most practical consequence of this stack is that you can leave
holes in your program and the type checker will tell you what to put
there.

```cure
fn safe_head(xs: List(T)) -> T = ?body
```

Compile, and Cure tells you:

```
?body : T
in scope:
  xs : List(T)
```

Multiple `??` holes are numbered `?_1`, `?_2`, ... in source order.
The REPL's `:holes` meta-command lists every hole recorded during the
last evaluation. This is how Idris users write programs; it is now how
Cure users can write programs.

## Totality, done gently

A function is **total** when it terminates on every input *and* its
pattern matching is exhaustive. Cure now classifies every function,
reports the result, but does not reject anyone by default. Opt in by
decorating with `#[total]`:

```cure
#[total]
fn factorial(n: Int) -> Int
  | 0 -> 1
  | n -> n * factorial(n - 1)
```

Direct structural recursion only in v0.17.0 -- mutual recursion joins
the party in v0.18.0.

## Polish

A dependent-type core is not enjoyable without a quality-of-life
layer around it. v0.17.0 brings:

- A real REPL (`:t`, `:doc`, `:effects`, `:load`, `:reload`, `:holes`,
  multi-line input, persisted history).
- A `cure watch` subcommand that recompiles, checks, or tests on
  every save with a 200 ms debounce.
- LSP inlay hints, signature help, formatting, rename with prepare,
  code lenses, semantic tokens, and workspace symbol search.
- An error catalog doubled in size (`E011`-`E020`) with
  `cure explain Eddd` explaining every entry.
- Three `cure new` templates (`--lib`, `--app`, `--fsm`), a
  `Cure.lock` lockfile, git-based dependencies, `cure deps tree`.
- A doctest harvester: any `##` block with `cure>` / `=>` pairs is
  a test case.
- A brand-new `docs/TUTORIAL.md` and `docs/DEPENDENT_TYPES.md`.

## Ecosystem groundwork

- MIT `LICENSE` file.
- Complete `mix.exs` `package` block; the Cure compiler is ready for
  Hex publication.
- `vscode-cure/` -- a TypeScript language-client scaffold, TextMate
  grammar, language configuration, README. Marketplace publication is
  deferred, but you can build it with `npm run compile` and press F5.
- `docs/PACKAGE_REGISTRY.md` -- the contract for the Cure registry we
  will start building in v0.18.0.

## What's next

v0.18.0, codenamed *Bring the Furniture*, lands the deferred "new
ideas": `proof` containers, `assert_type`, default record fields, the
finished `@derive`, property-based testing in `Std.Test`, `Std.Iter`,
mutual-recursion totality, and the first half of the package
registry.

The rewrite is complete. The type system is honest. The tools work.
Time to use the language.
